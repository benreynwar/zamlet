VECTOR MASK OPERATIONS
======================

Spec Reference: riscv-isa-manual/src/v-st-ext.adoc
Lines: 4037-4422

Overview
--------
Vector mask instructions operate on mask registers where each element is a single
bit. These instructions help manipulate predicate masks for conditional execution.

Key behaviors:
- Operate on single vector registers regardless of LMUL
- Always unmasked (vm=0 encodings reserved)
- Elements < vstart unchanged, vstart reset to zero after execution
- Tail elements updated with tail-agnostic policy
- Most instructions require vstart == 0 (raise illegal instruction exception otherwise)

-------------------
Repository: riscv-vector-tests
Test configs: 16 mask operation tests (8 logical ops, vcpop, vfirst, vmsbf,
vmsif, vmsof, viota, vid). See test_suite_reference.txt for complete list

Implementation Summary
----------------------
Currently implemented: 14 instructions (all 8 mask-mask logicals + VID.V +
vcpop.m + vfirst.m + vmsbf.m + vmsif.m + vmsof.m). Only viota.m remains.
Total in spec: 16 mask instructions
Coverage: ~88%

All 8 mask-mask logicals are unified under a single VmLogicMm class
(instructions/vector.py:865) parameterised by a VmLogicOp enum
(kamlet/kinstructions.py:97). The kinstr is VmLogicMmOp
(kinstructions.py:816) with the op evaluator at kinstructions.py:788
(_vmlogic_eval). Decoding dispatch lives in decode.py:948-970.

MASK-REGISTER LOGICAL OPERATIONS (Status: ✓)
=====================================================

Spec: v-st-ext.adoc lines 4043-4128

These operate on mask registers with 1-bit elements. All 8 instructions together
can generate any of the 16 possible binary logical functions. All 8 share the
VmLogicMm instruction class and VmLogicMmOp kinstr; only the VmLogicOp enum
value and decode dispatch differ between them.

VMAND.MM (Status: ✓)
  Spec: v-st-ext.adoc line 4060
  Encoding: funct6=0x19, funct3=OPMVV (0x2)
  Format: vmand.mm vd, vs2, vs1
  Implementation: VmLogicMm / VmLogicOp.AND
  Description: vd.mask[i] = vs2.mask[i] && vs1.mask[i]
  Notes: Pseudoinstruction: vmmv.m vd,vs = vmand.mm vd,vs,vs (copy mask).

VMNAND.MM (Status: ✓)
  Spec: v-st-ext.adoc line 4061
  Encoding: funct6=0x1d, funct3=OPMVV
  Format: vmnand.mm vd, vs2, vs1
  Implementation: VmLogicMm / VmLogicOp.NAND
  Description: vd.mask[i] = !(vs2.mask[i] && vs1.mask[i])
  Notes:
    - Pseudoinstruction: vmnot.m vd,vs = vmnand.mm vd,vs,vs (invert mask)
    - Used in conditional test

VMANDN.MM (Status: ✓)
  Spec: v-st-ext.adoc line 4062
  Encoding: funct6=0x18, funct3=OPMVV
  Format: vmandn.mm vd, vs2, vs1
  Implementation: VmLogicMm / VmLogicOp.ANDN
  Description: vd.mask[i] = vs2.mask[i] && !vs1.mask[i]
  Notes: Old mnemonic vmandnot can be retained as alias.

VMXOR.MM (Status: ✓)

  Implementation: VmLogicMm / VmLogicOp.XOR
  Description: vd.mask[i] = vs2.mask[i] ^^ vs1.mask[i]
  Notes: Pseudoinstruction: vmclr.m vd = vmxor.mm vd,vd,vd (clear mask).

SET-BEFORE/INCLUDING/ONLY-FIRST (Status: ✓)
============================================

Spec: v-st-ext.adoc lines 4185-4300

These instructions scan for the first set bit and set/clear surrounding bits.

VMSBF.M (Status: ✓)
  Spec: v-st-ext.adoc lines 4188-4227
  Encoding: funct6=0x14, vs1=0x01, funct3=OPMVV
  Format: vmsbf.m vd, vs2, vm
  Implementation: VmsFirstMask in instructions/vector.py:1540 (mode=BEFORE);
    SetMaskBits / SetMaskBitsMode kinstr in kamlet/kinstructions.py;
    decode at decode.py:994
  Description: Set all bits before first set bit
  Notes:
    - Writes 1 to elements before first 1, then 0 to that element and all following
    - If no 1 found, all active elements written with 1
    - vstart must be 0
    - vd cannot overlap vs2 or v0 (if masked)
    - Test file `tests/test_vms_first_mask.py` is missing on disk despite the

  Encoding: funct6=0x14, vs1=0x10, funct3=OPMVV
  Format: viota.m vd, vs2, vm
  Implementation: Not implemented

  Notes:

IMPLEMENTATION INFRASTRUCTURE
==============================

Existing Support:
- VmLogicMm class in instructions/vector.py:865 — unified class covering all 8
  mask-mask logical operations, parameterised by a VmLogicOp enum value
- VmLogicMmOp kinstr in kinstructions.py:816 with _vmlogic_eval evaluator at
  kinstructions.py:788
- Decoding for all 8 mask logicals in decode.py:948-970
- Vid class (instructions/vector.py:2009) and VidOp kinstr for vid.v
- ew=1 mask-vline iteration pattern established in VmLogicMm (iterates
  n_vlines = ceil(vl / (word_bytes * j_in_l * 8)) jamlet-wide vlines, asserting
  vs1/vs2 ordering ew=1)

What's Missing:
- Class for viota (parallel prefix sum). vcpop / vfirst / vmsbf / vmsif /
  vmsof are now in (commits 4d28467 + 65e1a59).
- Cross-jamlet parallel prefix-sum primitive (PrefixSumRound kinstr +
  WaitingPrefixSumRound transaction) — needed for viota and downstream
  vcompress.
- vstart==0 checking (most mask operations require vstart==0 except the
  logicals, which allow non-zero vstart via prestart preservation)
- Overlap checking (vd cannot overlap vs2/v0 for vmsbf, vmsif, vmsof, viota)

Summary
-------
14 of 16 instructions implemented:
- ✓ All 8 mask-mask logicals (vmand/vmandn/vmor/vmorn/vmxor/vmnand/vmnor/vmxnor)
  unified under VmLogicMm, including the standard pseudos (vmmv, vmclr, vmset,
  vmnot)
- ✓ vid.v (element index)
- ✓ vcpop.m (population count to x register) — J2J reduction tree
- ✓ vfirst.m (find-first-set to x register) — sync-network MIN
- ✓ vmsbf.m, vmsif.m, vmsof.m (set-before/including/only-first) — share
  VmsFirstMask + SetMaskBits kinstr
- ✗ viota.m (parallel prefix sum, used for compress) — needs PrefixSumRound

Mask operations are critical for:
- Conditional execution and predication
- Dynamic loop control (vfirst for early exit)
- Vector compress/expand (viota + indexed stores)
- Mask manipulation and combination
- Efficient predicate computation

Priority for implementation:
1. vcpop.m — unblocks FFT N=32 (current focus)
2. vfirst.m — likely next FFT blocker; shares funct6=0x10/OPMVV dispatch with
   vcpop.m (just a different vs1 encoding)
3. viota.m — essential for compress operations
4. vmsbf/vmsif/vmsof — specialised scan-first ops
