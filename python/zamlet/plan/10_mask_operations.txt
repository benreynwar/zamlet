VECTOR MASK OPERATIONS
======================

Spec Reference: riscv-isa-manual/src/v-st-ext.adoc
Lines: 4037-4422

Overview
--------
Vector mask instructions operate on mask registers where each element is a single
bit. These instructions help manipulate predicate masks for conditional execution.

Key behaviors:
- Operate on single vector registers regardless of LMUL
- Always unmasked (vm=0 encodings reserved)
- Elements < vstart unchanged, vstart reset to zero after execution
- Tail elements updated with tail-agnostic policy
- Most instructions require vstart == 0 (raise illegal instruction exception otherwise)

Implementation Summary
----------------------
Currently implemented: 14 instructions (all 8 mask-mask logicals + vcpop.m,
vfirst.m, vmsbf.m, vmsif.m, vmsof.m, vid.v)
Total in spec: 16 mask instructions
Coverage: 88% (viota.m deferred — see VECTOR IOTA section)

All 8 mask-mask logicals are unified under a single VmLogicMm class
(instructions/vector.py:894) parameterised by a VmLogicOp enum
(kamlet/kinstructions.py). The kinstr is VmLogicMmOp with the op evaluator
_vmlogic_eval. Decoding dispatch lives in decode.py.

vcpop.m / vfirst.m / vmsbf.m / vmsif.m / vmsof.m share infrastructure built
on top of the cross-jamlet ReduceSync transaction
(transactions/reduce_sync.py) and the per-jamlet MaskPopcountLocal /
SetMaskBits kinstrs (kamlet/kinstructions.py). vcpop.m additionally uses
the existing tree-reduction SUM path (oamlet/reduction.py).

MASK-REGISTER LOGICAL OPERATIONS (Status: ✓)
=====================================================

Spec: v-st-ext.adoc lines 4043-4128

These operate on mask registers with 1-bit elements. All 8 instructions together
can generate any of the 16 possible binary logical functions. All 8 share the
VmLogicMm instruction class and VmLogicMmOp kinstr; only the VmLogicOp enum
value and decode dispatch differ between them.

VMAND.MM (Status: ✓)
  Spec: v-st-ext.adoc line 4060
  Encoding: funct6=0x19, funct3=OPMVV (0x2)
  Format: vmand.mm vd, vs2, vs1
  Implementation: VmLogicMm / VmLogicOp.AND
  Description: vd.mask[i] = vs2.mask[i] && vs1.mask[i]
  Notes: Pseudoinstruction: vmmv.m vd,vs = vmand.mm vd,vs,vs (copy mask).

VMNAND.MM (Status: ✓)
  Spec: v-st-ext.adoc line 4061

  Implementation: VmLogicMm / VmLogicOp.NAND
  Description: vd.mask[i] = !(vs2.mask[i] && vs1.mask[i])
  Notes:

SET-BEFORE/INCLUDING/ONLY-FIRST (Status: ✓)
============================================

Spec: v-st-ext.adoc lines 4185-4300

These instructions scan for the first set bit and set/clear surrounding bits.
All three are unified under VmsFirstMask (instructions/vector.py:1538),
parameterised by a SetMaskBitsMode (kinstructions.py:113). The dispatch
runs ReduceSync(MIN_EL_INDEX, ew=1) into a vector tmp word and then
SetMaskBits (kinstructions.py:1057) with the appropriate mode.

VMSBF.M (Status: ✓)
  Spec: v-st-ext.adoc lines 4188-4227
  Encoding: funct6=0x14, vs1=0x01, funct3=OPMVV
  Format: vmsbf.m vd, vs2, vm
  Implementation: VmsFirstMask with SetMaskBitsMode.LT (decode.py:996)
  Description: Set all bits before first set bit
  Notes:
    - Writes 1 to elements before first 1, then 0 to that element and all following
    - If no 1 found, all active elements written with 1 (sentinel 0xFFFFFFFF
      makes the unsigned LT compare true everywhere)
    - vstart must be 0
    - vd cannot overlap vs2 or v0 (if masked)

VMSIF.M (Status: ✓)
  Spec: v-st-ext.adoc lines 4232-4263
  Encoding: funct6=0x14, vs1=0x03, funct3=OPMVV
  Format: vmsif.m vd, vs2, vm
  Implementation: VmsFirstMask with SetMaskBitsMode.LE (decode.py:1006)
  Description: Set all bits before and including first set bit
  Notes:
    - Like vmsbf but also includes the first 1
    - vstart must be 0
    - vd cannot overlap vs2 or v0 (if masked)

VMSOF.M (Status: ✓)
  Spec: v-st-ext.adoc lines 4268-4300
  Encoding: funct6=0x14, vs1=0x02, funct3=OPMVV
  Format: vmsof.m vd, vs2, vm
  Implementation: VmsFirstMask with SetMaskBitsMode.EQ (decode.py:1001)
  Description: Set only the first set bit
  Notes:
    - Only sets the first 1, clears all other bits
    - If no 1 found, all active destination bits cleared (sentinel
      0xFFFFFFFF cannot equal any real element index)
    - vstart must be 0
    - vd cannot overlap vs2 or v0 (if masked)

VIOTA.M (Status: ✗ — deferred)
  Spec: v-st-ext.adoc lines 4315-4355
  Encoding: funct6=0x14, vs1=0x10, funct3=OPMVV
  Format: viota.m vd, vs2, vm
  Implementation: Not implemented (deferred)
  Description: Parallel prefix sum of mask values
  Deferral rationale: viota.m is a parallel-prefix-sum primitive. Unlike the
    other mask ops, it cannot be expressed via the existing ReduceSync /
    MaskPopcountLocal / SetMaskBits primitives — it needs a true scan
    (intra-jamlet local scan + cross-jamlet prefix-sum). Its only critical
    consumer is vcompress.vm (a separate kinstr, also deferred) plus
    compiler autovectorisation of compress-like patterns. Will be
    revisited if/when vcompress is prioritised. See PLAN_mask_ops.md
    "Deferred / future work".
  Notes:
    - For each element, writes sum of all bits with lower index
    - Result zero-extended if SEW > result width
    - Least-significant SEW bits retained if overflow
    - Can be masked (only enabled elements contribute)
    - vstart must be 0
    - vd cannot overlap vs2 or v0 (if masked)
    - Used with indexed stores for vector compress (see spec example)

IMPLEMENTATION INFRASTRUCTURE
==============================

Existing Support:
- VmLogicMm class in instructions/vector.py:894 — unified class covering all 8
  mask-mask logical operations, parameterised by a VmLogicOp enum value
- VmLogicMmOp kinstr in kinstructions.py with _vmlogic_eval evaluator
- Decoding for all 8 mask logicals in decode.py
- Vid class (instructions/vector.py:2409) and VidOp kinstr for vid.v
- ew=1 mask-vline iteration pattern established in VmLogicMm (iterates
  n_vlines = ceil(vl / (word_bytes * j_in_l * 8)) jamlet-wide vlines, asserting
  vs1/vs2 ordering ew=1)
- ReduceSync transaction (transactions/reduce_sync.py:38) — cross-jamlet
  idempotent reduction across the sync network. Supported ops include AND,
  OR, MIN/MAX (signed/unsigned), MIN_EL_INDEX, MAX_EL_INDEX over
  ew ∈ {1,8,16,32}. Result delivered to every jamlet and the lamlet.
  MIN_EL_INDEX returns the global RVV element_index of the first set bit
  (sentinel 0xFFFFFFFF when no bit is set).
- MaskPopcountLocal kinstr (kinstructions.py:990) — per-jamlet popcount of
  active mask bits, ew=1 → ew=32. Used by vcpop.m.
- SetMaskBits kinstr (kinstructions.py:1057) — reads a vreg where every
  word holds the same value (e.g. the MIN_EL_INDEX result) and writes a
  destination mask register with bit i = (i CMP value) where CMP is one of
  LT / LE / EQ (SetMaskBitsMode). Used by vmsbf/vmsif/vmsof.

What's Missing:
- viota.m (parallel prefix sum) — deferred. Requires a true scan primitive
  (intra-jamlet local scan + cross-jamlet prefix sum). Only critical for
  vcompress.vm (also deferred) and compiler autovectorisation of
  compress-like patterns. See PLAN_mask_ops.md "Deferred / future work".
- vcompress.vm — separate from this plan; tracked elsewhere.

Summary
-------
9 of 16 instructions implemented:
- ✓ All 8 mask-mask logicals (vmand/vmandn/vmor/vmorn/vmxor/vmnand/vmnor/vmxnor)
  unified under VmLogicMm, including the standard pseudos (vmmv, vmclr, vmset,
  vmnot)
- ✓ vid.v (element index)
- ✗ vcpop.m (population count to x register) — immediate FFT N=32 blocker
- ✗ vfirst.m (find-first-set to x register) — sibling at vs1=0b10001
- ✗ vmsbf.m, vmsif.m, vmsof.m (set-before/including/only-first)
- ✗ viota.m (parallel prefix sum, used for compress)

Mask operations are critical for:
- Conditional execution and predication
- Dynamic loop control (vfirst for early exit)
- Vector compress/expand (viota + indexed stores)
- Mask manipulation and combination
- Efficient predicate computation

Priority for implementation:
1. vcpop.m — unblocks FFT N=32 (current focus)
2. vfirst.m — likely next FFT blocker; shares funct6=0x10/OPMVV dispatch with
   vcpop.m (just a different vs1 encoding)
3. viota.m — essential for compress operations
4. vmsbf/vmsif/vmsof — specialised scan-first ops
