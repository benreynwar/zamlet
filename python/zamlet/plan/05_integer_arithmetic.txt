VECTOR INTEGER ARITHMETIC INSTRUCTIONS
=======================================

Spec Reference: riscv-isa-manual/src/v-st-ext.adoc
Lines: 2367-3046

Overview
--------
Vector integer arithmetic provides add, subtract, multiply, divide, shifts,
bitwise operations, comparisons, and min/max operations on integer elements.
Most operations support .vv (vector-vector), .vx (vector-scalar), and .vi
(vector-immediate) forms. Unless stated, operations wrap on overflow.

Note: This file covers SINGLE-WIDTH operations only. Widening/narrowing operations
are in 06_integer_widening_narrowing.txt.

External Test Suite
-------------------
Repository: riscv-vector-tests
Test configs: 77 integer arithmetic tests covering add/sub, bitwise, shifts,
compares, min/max, multiply, divide, carry/borrow, merge/move operations
See test_suite_reference.txt for complete list

Implementation Summary
----------------------
Currently implemented: ~48 instructions (ADD, SUB, RSUB, AND, OR, XOR, SLL, SRL, SRA,
MUL, MACC, MADD, NMSAC, NMSUB across VV/VX/VI forms; MIN, MINU, MAX, MAXU across
VV/VX forms; comparisons; moves; merge)
Total in spec: ~70 single-width integer arithmetic instructions
Coverage: ~68%

VMULH.VV, VMULH.VX (Status: ✓)
  Description: Signed multiply, return high SEW bits of product
  Implementation: VArithVv/VArithVx with VArithOp.MULH; high half computed
    in _compute_arith via (src1*src2) >> (eb*8). Routed through the simple
    same-width path (both sources signed).
  Decoding: decode.py funct6=0x27 funct3=0x2 (.vv) / 0x6 (.vx)

VMULHU.VV, VMULHU.VX (Status: ✓)
  Description: Unsigned multiply, return high SEW bits
  Implementation: VArithVv/VArithVx with VArithOp.MULHU. Same compute path
    as MULH; SIGNED_INT_OPS / UNSIGNED_INT_OPS membership selects the
    unpack format (unsigned for MULHU).
  Decoding: decode.py funct6=0x24 funct3=0x2 (.vv) / 0x6 (.vx)

VMULHSU.VV, VMULHSU.VX (Status: ✓)
  Description: Signed-Unsigned multiply, return high bits (vs2 signed,
    vs1/rs1 unsigned)
  Implementation: VArithVvOv/VArithVxOv with VArithOp.MULHSU and
    shape=OvShape.SAME (a new same-width shape added so the per-source
    signedness path can carry src1_signed=False, src2_signed=True with
    src/dst all at SEW). High half computed in _compute_arith.
  Decoding: decode.py funct6=0x26 funct3=0x2 (.vv) / 0x6 (.vx)

VADC.VVM (Status: ✓)
  Description: Add with carry-in from v0
  Implementation: VArithVv with VArithOp.ADC. Routed via CARRY_BORROW_OPS
    in instructions/vector.py:VArithVv.update_state, which forces mask_reg=None
    (full writes) and carry_in_reg=0 (v0 supplies the per-element carry-in).
    VArithVvOp threads carry_in_preg through admit/execute and _compute_arith
    returns src2_val + src1_val + carry_in.
  Decoding: decode.py funct6=0x10 funct3=0x0 vm==0
  Notes: vm=0 mandatory (vm=1 reserved). Every body element is written.

VADC.VXM (Status: ✓)
  Description: Add with carry-in from v0 (scalar)
  Implementation: VArithVx with VArithOp.ADC. CARRY_BORROW_OPS membership
    drives the wrapper to set carry_in_reg=0 / mask_reg=None. VArithVxOp
    threads carry_in_preg through admit/execute the same way as VArithVvOp.
  Decoding: decode.py funct6=0x10 funct3=0x4 vm==0
  Notes: vm=0 mandatory (vm=1 reserved).

VADC.VIM (Status: ✓)
  Description: Add with carry-in from v0 (5-bit immediate)
  Implementation: VArithVi with VArithOp.ADC. The immediate is sign-extended
    to word_bytes by VArithVi.update_state and dispatched through the same
    VArithVxOp path used by .vxm.
  Decoding: decode.py funct6=0x10 funct3=0x3 vm==0
  Notes: vm=0 mandatory (vm=1 reserved).

VMADC.VVM, VMADC.VXM, VMADC.VIM (Status: ✗)
  Description: Add and generate carry-out mask
  Notes: Can be masked or unmasked. Phase D — VCmpOp surgery.

VMADC.VV, VMADC.VX, VMADC.VI (Status: ✗)
  Description: Add and generate carry-out (no carry-in)

VSBC.VVM (Status: ✓)
  Description: Subtract with borrow-in from v0
  Implementation: VArithVv with VArithOp.SBC. Same plumbing as VADC.VVM —
    CARRY_BORROW_OPS membership in kinstructions.py drives the wrapper to
    set carry_in_reg=0 and mask_reg=None. _compute_arith returns
    src2_val - src1_val - carry_in.
  Decoding: decode.py funct6=0x12 funct3=0x0 vm==0
  Notes: vm=0 mandatory (vm=1 reserved). No .vim form (reserved).

VSBC.VXM (Status: ✓)
  Description: Subtract with borrow-in from v0 (scalar)
  Implementation: VArithVx with VArithOp.SBC. Same dispatch as VADC.VXM.
  Decoding: decode.py funct6=0x12 funct3=0x4 vm==0
  Notes: vm=0 mandatory. No .vim form (reserved by spec).

Summary
-------
Basic integer arithmetic support is minimal. Only 5 operations are implemented:
- vadd.vx (vector-scalar add)
- vmul.vv/vx (vector multiply)
- vmacc.vv/vx (multiply-accumulate)
- vmsle.vi (comparison)
- vmv.* (move operations)

Missing ~65 instructions including:
- Most add/subtract variants
- All bitwise operations (and, or, xor)
- All shift operations
- Most comparison operations
- All min/max operations
- Divide and remainder
- High multiply variants
- Carry/borrow operations
- Merge operations

Priority for implementation:
1. Basic operations (sub, bitwise, shifts) - most commonly used
2. Comparisons - needed for conditional code
3. Min/max - common in many algorithms
4. Divide/remainder - less common but sometimes necessary
5. Carry/borrow - specialized multi-word arithmetic
