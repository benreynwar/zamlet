WIDENING AND NARROWING INTEGER OPERATIONS
==========================================

Spec Reference: riscv-isa-manual/src/v-st-ext.adoc
Lines: 2396-2432 (Widening add/subtract)
Lines: 2433-2462 (Integer extension)
Lines: 2619-2647 (Narrowing shifts)
Lines: 2912-2930 (Widening multiply)
Lines: 2963-2986 (Widening multiply-add)

Overview
--------
Widening operations take SEW-bit operands and produce 2*SEW-bit results.
Narrowing operations take 2*SEW-bit operands and produce SEW-bit results.
These are essential for mixed-precision arithmetic, avoiding overflow, and
implementing larger integer operations.

Extension operations zero- or sign-extend narrow values to wider destinations
without arithmetic.

-------------------

Implementation Summary
----------------------
Currently implemented: 6 extensions + 30 widening/narrowing arith
Total in spec: ~40 widening/narrowing instructions
Coverage: ~90%

Implementation: the width-asymmetric arithmetic family (widening add/sub/mul/
mac and narrowing right shifts) is handled by `VArithVvOv` / `VArithVxOv` in
python/zamlet/instructions/vector.py, which lower to `VArithVvOvOp` /
`VArithVxOvOp` in python/zamlet/kamlet/kinstructions.py. The wrapper carries
an `OvShape` enum (BASE = both srcs at SEW; WIDE = src2 at 2*SEW; NARROW =
src2 at 2*SEW, dst at SEW) and per-source signedness flags; the kinstruction
unpacks each operand at its own width, computes via `_compute_arith` (with
the shift mask widened via `shift_eb` for narrowing shifts), and packs the
destination. Decoder entries live in python/zamlet/decode.py under the
opcode==0x57 block, funct6 0x2c–0x2f (narrowing) and 0x30–0x3f (widening).

NARROWING SHIFTS (Status: ✓)
=============================

Spec: v-st-ext.adoc lines 2619-2647

Take 2*SEW source, shift right, store SEW-bit result. Essentially combines
shift and narrowing into one operation. Implemented through VArithVvOv /
VArithVxOv with shape=NARROW. The shift count is masked to lg2(2*SEW) bits
via the `shift_eb` parameter on `_compute_arith` (the kinstruction passes
src2_eb = 2*SEW so the mask is widened from the dst width).

VNSRL.WI, VNSRL.WV, VNSRL.WX (Status: ✓)
  Spec: v-st-ext.adoc line 2631-2633
  Format: vnsrl.wi vd, vs2, uimm, vm / vnsrl.wv vd, vs2, vs1, vm /
          vnsrl.wx vd, vs2, rs1, vm
  Implementation: VArithVvOv / VArithVxOv with op=SRL, shape=NARROW, both
    sources unsigned (src2 is the wide value, src1/scalar is the count).
    Decoder: funct6=0x2c, funct3=0x0 (.wv) / 0x3 (.wi, is_imm=True) /
    0x4 (.wx).
  Description: Narrowing logical shift right, SEW = (2*SEW) >> shift
  Notes: Pseudoinstruction vncvt.x.x.w vd,vs,vm = vnsrl.wx vd,vs,x0,vm
    (narrow without shift) is supported via the same path.

REGISTER CONSTRAINTS
====================

Widening Operations:
  - Destination vd has EEW=2*SEW, EMUL=2*LMUL
  - Source operands (for SEW+SEW→2*SEW) have EEW=SEW, EMUL=LMUL
  - For 2*SEW+SEW→2*SEW (.w forms), first source is 2*LMUL
  - vd cannot overlap source register groups

Narrowing Operations:
  - Source vs2 has EEW=2*SEW, EMUL=2*LMUL
  - Destination vd has EEW=SEW, EMUL=LMUL
  - vd cannot overlap vs2

Extension Operations:
  - Source EMUL = (source_EEW / SEW) * LMUL
  - For vf2: source EMUL = LMUL/2
  - For vf4: source EMUL = LMUL/4
  - For vf8: source EMUL = LMUL/8

Remaining work
--------------
1. Narrowing saturating shifts (vnclip / vnclipu, funct6 0x2e/0x2f).
   Distinct from the SRL/SRA path because the destination is saturated to
   the signed/unsigned range and vxsat is updated. Requires new
   VArithOp.NCLIP / VArithOp.NCLIPU values plus saturation logic in
   _compute_arith / _ov_truncate_int. Out of scope for the FFT push.

2. Overlap / register-group constraint checking.
   Spec forbids vd overlapping the source group when the widths differ
   (widening vd is 2*LMUL, .w form has src2 also 2*LMUL, narrowing vd
   cannot overlap vs2). Currently the kinstruction admits and renames
   without checking these constraints.

3. Dedicated unit tests for the Ov classes (8→16, 16→32, 32→64 widening;
   16→8, 32→16, 64→32 narrowing; mixed-signedness mul/mac variants;
   varying LMUL).

Summary
-------
Width-asymmetric integer add/sub/mul/mac and narrowing right shifts are
implemented through the VArithVvOv / VArithVxOv classes (see top of file
for the lowering chain). Saturating narrowing (vnclip/vnclipu) and overlap
checking remain.

Implemented (this round): 30 instructions
  - 16 widening add/subtract variants (BASE + WIDE shapes, .vv/.vx, signed
    and unsigned)
  - 6 widening multiply variants (vwmul / vwmulu / vwmulsu, .vv/.vx)
  - 7 widening multiply-add variants (vwmaccu, vwmacc, vwmaccsu .vv/.vx
    plus vwmaccus .vx-only)
  - 6 narrowing right-shift variants (vnsrl/vnsra .wv/.wx/.wi)

Still unimplemented: ~4 narrowing-saturating-shift instructions
  (vnclip, vnclipu, .wv/.wx/.wi forms collapsed under each).
