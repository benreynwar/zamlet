Key Guidelines (IMPORTANT!)
---------------------------
1. Spec paths: Use "riscv-isa-manual/src/..." NOT "/home/ben/Code/..."
2. Code references: Use class/method names or search patterns, NOT line numbers
3. Encodings: Get from spec .edn files (vmem-format.edn, etc), NOT Python code
4. After each file: Update spec_index.txt to point sections to new plan file
5. Keep updating this plan_plan.txt with useful info as you discover it!
6. Test references: Use filename only (e.g., "vadd.vv.toml"), full catalog in test_suite_reference.txt

Process
-------
1. Read existing plan files to understand the format before creating new ones
2. Read spec (use spec_index.txt for line numbers)
3. Search implementation: Grep for classes, check decode.py by opcode/pattern
4. Check tests in tests/ directories
5. Document with ✓ (done), ⚠ (partial), ✗ (not done)
6. Update spec_index.txt and this file's progress counter
